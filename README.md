# kniho-hlod-backend

REST API backend for the **kniho-hlod** book lending application.

## Tech Stack

- **Runtime:** Node.js 22 + TypeScript
- **Framework:** Express (via `@eleansphere/be-core`)
- **ORM:** Sequelize 6 + PostgreSQL (`pg`)
- **Auth:** JWT
- **Password hashing:** bcrypt
- **File uploads:** multer (avatar images stored as BLOB)
- **Database:** Supabase (PostgreSQL)
- **Deployment:** Render (triggered by GitHub Actions)

## Architecture

The app is built on the internal `@eleansphere/be-core` framework, which handles Express setup, database connection, CRUD routing, file uploads, and JWT authentication. Domain models and DTO types come from `@eleansphere/kniho-hlod-service`.

```
src/
├── index.ts          # Entry point — calls createApp() with config
├── plugin.ts         # Custom plugin — registers User model and routes
└── models/
    ├── user.ts       # User model (extends CoreEntity)
    └── profile-images.ts  # ProfileImage model
```

## API Endpoints

Endpoints are auto-generated by `be-core` from model configs, plus custom routes in `plugin.ts`.

## Local Development

### Prerequisites

- Node.js 22+
- Access to the `@eleansphere` GitHub Package Registry (requires a GitHub PAT with `read:packages`)

## Deployment

Pushes to `master` trigger the GitHub Actions workflow (`.github/workflows/deploy.yml`), which:

1. Checks out code
2. Installs dependencies from GitHub Package Registry
3. Compiles TypeScript
4. Triggers a Render deploy webhook

Required GitHub repository secrets:

- `NODE_AUTH_TOKEN` — GitHub PAT with `read:packages`
- `RENDER_DEPLOY_HOOK_URL` — Render deploy hook URL

## Environment Variables

| Variable          | Required   | Description                              |
| ----------------- | ---------- | ---------------------------------------- |
| `DATABASE_URL`    | Yes        | PostgreSQL connection string             |
| `JWT_SECRET`      | Yes        | Secret for signing JWT tokens            |
| `PORT`            | No         | Server port (default: 3000)              |
| `NODE_AUTH_TOKEN` | Build only | GitHub token for `@eleansphere` packages |
